Operating Systems Terminology
Term Definition
Operating System A system that maintains the functions of a computer such as time-sharing, maintaining programs and data files, and responding to unser requests (p. 153).
Job queue The way in which the execution of a program (Jobs) is ordered depending on their priority (p. 151)
FIFO First in first out (p. 151).
Interactive Processing Allows communication between the User and the program using a remote terminal (p. 152)
Time-Sharing a feature that allows operating systems to serve multiple users at the same time (p. 153)
Multiprogramming Is a means to allow time-sharing by dividing jobs into intervals where only one job is allowed to be executed at the time of its interval (p. 153).
Application Software Are programs that performs tasks depending on the machine's purpose (p.155).
System Software Performs tasks that are general to computer systems (p. 156).
Utility Software Extends the abilities of an operating system (p. 156)
Graphical user interface (GUI) In which the files and programs to be manipulated are displayed as icons (p. 157).
Shell A type of user interface that communicated with users, using a keyboard and monitor, through textual messages (p. 157).
Kernel The internal part of an operating system that contains software components that perfomrs funtions for computer installation (p. 158).
Device driver A component of a kernel that communicates with controllers or sometimes directly with peripheral devices to carry out operations on the peripheral devices of their machines (p. 159).
File manager Allows the user to use the machines storage facilities (p. 158).
Memory manager Coordinates the machines use of memory (p. 160).
Virtual memory The memory created by paging (uniformly-sized units of data that the memory manager shuffles between the main memory and mass storage) (p. 160).
Boot strapping A procedure performed by the computer everytime it turns on where the operating system in transfered from mass storage into main memory (p. 160).
ROM Read-only memory. Memory that can only be read and not changed (p. 161).
Deadlock When multiple processes are blocked because they are each waiting for a resource that is being used by the other (p. 168).
Spooling Holding data for output at a more convenient time (p. 170).
Super user (or administrator) One who can alter the operating system among other privileges in ways a normal user can not.
Multi-factor authentification Is used for security. It requires more than one piece of evidence for access to be granted to the device (p. 175).
Source: Brookshear, Glenn J, and Dennis Brylow. Computer Science: An Overview. 13th ed., Pearson, https://www.pearson.com/, Accessed 29 Mar. 2023.
