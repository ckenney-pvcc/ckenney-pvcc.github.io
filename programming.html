Programming Algorithms & Software Testing

Phases of computer program developement:

  • Phase 1: Understand the problem
  • Phase 2: Design an algorithm to solve the problem
  • Phase 3: Write the program code in appropriate language based on the algorithm
  • Phase 4. Enter the program and save
  • Phase 5: Execute the program
  • Phase 6: Test the program, find errors and debug, then retest
  • Phase 7: Document and evaluate the program

Programming Structures

Sequential structure: Top-down execution
Decision structure: if/then/else
Iterative(Looping):
  • while loop: continue looping as long as something is true
  • repeat/until loop: continue looping until something is true
  • for loop: loop a given number of times
Recursive structures:
  • Repeat a set of instruction as a subtask of itself
  • Repitition continues until:
    • degenerative case, which is a termination condition, is met
    • (degenerative case is also called a "base case")

Two classic programming/database problems

Sorting: Descending order/Ascending order
  • Sorting numbers
    • Sorting strings (in accordance with ASCII values)
  • Searching: find specific data that meet a certain criteria
    • In database:
      • Filters: temporary search
      • Queries: search criteria are saved so they don't have to be recreated

Two popular search techniques:

Sequential search: search from top to bottom or bottom to top, one item at a time
Binary search:
  • only works if data is SORTED first
  • start in the middle, then go to half of that, ect.

Two classic program goals:

Efficiency: Program takes minimal time and/or resources
Correctness: Program runs in expected ways and works for all expected conditions
Software testing and verification
Term Definition
Acceptance testing: make sure the system works like it's supposed to
Integration testing make sure software components or functions work together
Unit testing Make sure that the smallest testable part of a application (also called a unit) works as intended
Functional testing Checks functions by using scenarios based on the functions it is supposed to perform.
Performance testing To test how a software works under different loads for example during real-life work load conditions.
Regression testing To test if new features will mess with how the software functions
Stress testing To test how much a system can be pressured before failing.
Usability testing To test how easily a customer can use the system/web application.
From: IBM What is software testing? https://www.ibm.com/topics/software-testing

Software testing is done to detect:

  • Architectural flaws
  • Poor design decisions
  • Invalid or incorrect functionality
  • Security vulnerabilities
  • Scalability issues
Testing best practices
Term Definition
Continuous testing Testing software as it is developed, so that software is tested early and there is less risks.
Configuration management Organizations keep test assets. They also track what parts of software to test. Teams have access to these test access such as code, test scripts and results.
Service virtualization Is a placeholder for service/systems that have not been developed yet to aide in teams testing sooner.
Defect or “bug tracking” Tracking issues in the code so that teams for testing and developement can improve the system.
Metrics and reporting Is important for teams to be able to see the progress in the developement in the project. Teams are able to share the current status of the project and their goals for the project
